🤖 Personal AI Agents

Users create personalized AI agents that learn their preferences, dietary restrictions, schedules, and relationships. These agents represent users in conversations with businesses and services.

🔄 Multi-Agent Orchestration

Proprietary bidding engine determines which specialist agents should respond to each query. Health agents handle dietary questions, finance agents track budgets, assistants coordinate logistics.

🏪 Agent Marketplace

Businesses publish their own AI agents. When users scan a QR code at a restaurant, their personal agents communicate with the restaurant's agent to handle reservations, dietary restrictions, and preferences.

🧠 Persistent Memory

The platform learns from every interaction. Facts about users are extracted and stored, creating an ever-improving profile that agents use to provide personalized, proactive service.

How a Conversation Flows

1
User Scans QR Code

iOS app captures QR, identifies the business's public agent

↓
2
WebSocket Connection Established

Real-time connection to Lambda via API Gateway WebSocket

↓
3
Orchestration Engine Activates

Bidding engine evaluates which of user's agents should participate

↓
4
Agents Communicate

User's agents share relevant info (dietary needs, preferences) with host agent

↓
5
Facts Extracted & Stored

AI extracts new facts from conversation, stores in user's persistent memory

↓
6
Task Completion Detected

If a reservation was made or question answered, task is automatically marked complete
